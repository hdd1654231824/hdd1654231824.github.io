一、标题：

Gitbash基础操作

二、简介

  • 关于Gitbash的基础知识和基本命令行的简单介绍

三、命令

  1. curl:curl 命令是一个在 Linux 系统中利用 URL 工作的命令行文件传输工具，常用于服务访问和文件下载。curl 支持 HTTP、HTTPS、FTP 等多种协议（默认是 HTTP 协议），可用于模拟服务请求以及上传和下载文件。
  2. tracert：简单网络诊断工具，探测数据包从源地址到目的地址经过的路由器IP地址
    3. (1)-d：表示不让 tracert 根据节点主机名查找路由的 IP 地址，直接进行路由跟踪。当路由器不支持 
ICMP/UDP/ICMPv6 数据包时，建议使用该选项。
(2)-h maximum_hops：指定最多经过多少个节点进行路由跟踪，默认值为 30。使用该选项可以更改该值。
(3)-j host-list：枚举一个节点列表，并在路由跟踪过程中只走该列表中的节点。
(4)-w timeout：设置等待每个回复消息的超时时间，默认值为 4000 毫秒。使用该选项可以更改该值。
(5)-R：指示 tracert 跟踪路由时，使用“参考路由”而不是“严格源路由”。如果启用了该选项，则可以跨越
某些无法到达的能够到达目标的路由器。
(6)-S srcaddr：指定源地址。如果在单个计算机上安装了多个网络适配器，则使用该选项可以指定从哪个适
配器发出路由跟踪请求。
(7)-4：强制使用 IPv4 进行路由跟踪。
(8)-6：强制使用 IPv6 进行路由跟踪。
(9)target_name：表示要查询的目标 IP 地址或主机名。
  3. ping：执行 ping 指令会使用 ICMP（Internet Control Message Protocol） 传输协议，发出要求回应的信息，若远端主机的网络功能没有问题，就会回应该信息，因而得知该主机运作正常。
    4. (1)-d 使用Socket的SO_DEBUG功能。
(2)-c <完成次数> 设置完成要求回应的次数。
(3)-f 极限检测。
(4)-i<间隔秒数> 指定收发信息的间隔时间。
(5)-I<网络界面> 使用指定的网络接口送出数据包。
(6)-l<前置载入> 设置在送出要求信息之前，先行发出的数据包。
(6)-n 只输出数值。
(7)-p<范本样式> 设置填满数据包的范本样式。
(8)-q 不显示指令执行过程，开头和结尾的相关信息除外。
(9)-r 忽略普通的Routing Table，直接将数据包送到远端主机上。
(10)-R 记录路由过程。
(11)-s<数据包大小> 设置数据包的大小。
(12)-t<存活数值> 设置存活数值TTL的大小。
(13)-v 详细显示指令的执行过程。
(14)-w  在 deadline 毫秒后退出。
(15)-W  在等待 timeout 毫秒后开始执行。
  4. wegt：用于下载文件
    5. (1)-O 选项以其他名称保存下载的文件
wget -O /home/ym/demo.tar url 
(2)-P 选项将文件下载到指定目录
wget -P /home/ym url 
(3)-c 选项断点续传,文件较大，网络断开未下载完成时，-c选项可以恢复下载，无需从头下载
wget -c url
(4)-b 选项在后台下载文件
wget -b url  #默认下载日志重定向到当前目录下wget-log文件中，使用`tail -f wget-log`查看
(5)-i 选项下载多个文件
#创建一个文本文件download_list.txt,将所有url添加到该文件，每个url必须单独一行
wget -i download_list.txt
(6)--limit-rate 选项限制下载速度，默认wget命令会以全速下载，但有时下载一个非常大的资源,
可能会占用大量的可用带宽，影响其他使用网络的任务，这时就要限制下载速度
wget --limit-rate=1m url  #下载速度限制为1m/s 
(7)-Q 选项限制总下载文件大小
wget -Q5m -i filelist.txt #下载的文件超过5M而退出下载，你可以使用。注意：这个参数对单个
文件下载不起作用，只能递归下载时才有效。
(8)--tries 选项增加重试次数，如果网络有问题或下载一个大文件有可能会下载失败，wget默认重
试20次，我们可以使用-tries选项来增加重试次数。
wget --tries=30 url
  5. cut:可以用来分割字符串，显示指定范围的内容，可以具体根据指定的字节范围、字符范围、分割字段范围来显示文本内容。还可以合并多个文件的内容。
    6. (1)-b	以字节为单位，显示行中指定字节范围的字符
(2)-c	以字符为单位，显示行中指定字符范围的字符
(3)-d	指定字段的分隔符，默认的字段分隔符为制表符“TAB”。字段分隔符必须加引号，单引号或者
双引号都可以，分隔符与选项 -d 之间允许有空格
(4)-f	显示指定字段的内容
(5)-n	与 -b 选项连用，不分割多字节字符
(6)--output-delimiter=	指定输出的字段分隔符，分隔符和选项之间不允许有空格，分隔符
可以不加引号，但是如果指定的分隔符是特殊符号必须加引号
(7)--help	显示指令的帮助信息
(8)--version	显示指令的版本信息
(9)--complement	用来排除指定范围的字符内容
  6. ^: 如grep '^tor' cet6.txt 查找tor开头的内容
  7. $:如grep 'tor$' cet6.txt 查找tor结尾的内容

四、总结

gitbash是一个在windows系统上运行Git命令的工具，通过使用命令行实现对计算机的操作，更加接近于计算机的底层。
